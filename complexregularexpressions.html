Complex regular expressions

Background

You've now seen the funamental operators used in building regular expressions. Now let's look at the more complex types of regular expressions that can be built up using those operators.

Using an operator multiple times

Operators can be used multiple times in the same expression.

Example

Σ = {a, b, c, d, e}

R = a ∪ b ∪ c ∪ d ∪ e

This regular expression recognizes the strings a, b, c, d and e.

Example

Σ = {a, b, c, d, e}

R = abcde

This regular expression recognizes the string abcde.

Note

While this is also true for the Kleene closure operator, I'll omit an example because, for any regular expression R, R** = R*.

Using multiple operators

More complex regular expressions can be built up using multiple operators.

Example

Σ = {a, b, c}

R = ab ∪ c*

This regular expression recognizes the strings ab, ε, c, cc, ccc, ...

Example

Σ = {a, b, c}

R = a(b ∪ c*)

This regular expresion recognizes the strings ab, a, ac, acc, accc, ...

Going deep with parentheses

Since there is no maximum depth to expressions in parentheses, any order of operations can be applied deep down into regular expressions.

Example

Σ = {a, b, c, d, e, f}

R = a(b ∪ c(d(e ∪ f)*)*)

Some examples of strings recognized by this regular expression:

  1. ab
  2. ac
  3. acd
  4. acde
  5. acdf
  6. acdee
  7. acdef
  8. acdeee
  9. acdedf
