Regular expressions

Background

In order to formally define a regular language in a concise way, we can use a regular expression.

Base symbols

We will start with declaring that, for any alphabet Σ, R = ε is a regular expression.

Similarly, we will declare that, for an σ ∈ Σ, R = σ is a regular expression.

Examples

Σ = {}

R = ε

This regular expression recognizes the empty string ε.

Σ = {a}

R = a

This regular expression recognizes the string a.

Concatenation of expressions

We want to allow for the inclusion of strings consisting of multiple symbols in our regular language. In order to support this, we want to allow the concatenation (squishing together) of two grammars to create a more complex grammar.

Example

Σ = {a, b}

We know that R1 = {a} and R2 = {b} are regular expressions.

R = R1R2 = ab

This regular expression recognizes the string ab.

Union of expressions

We want to allow for the choice of two expressions in our regular language. In order to support this, we want to allow the union ("or" operation) of two grammars to create a more complex grammar.

Example

Σ = {a, b}

We know that R1 = {a} and R2 = {b} are regular expressions.

R = R1 ∪ R2 = a ∪ b

This regular expression recognizes the strings a and b.

Kleene closure of an expression

We want to allow the repretition of expressions in our regular language. In order to support this, we want to allow the Kleene closure (*) operator to be applied to a grammar to create a more complex grammar.

Σ = {a}

We know that R1 = {a} is a regular expression.

R = R1*

This regular expression recognizes the strings ε, a, aa, ..., with no limit on the string length.
